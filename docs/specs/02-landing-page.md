# Feature: 랜딩 페이지

## 목적
- 사용자가 서비스에 처음 진입했을 때 보는 메인 화면
- AI 추천 또는 직접 만들기 두 가지 진입 경로 제공
- 모던하고 깔끔한 첫인상

## 상세 요구사항
- [ ] 헤더: 로고 + 서비스명 + 우측 상단 로그인 버튼 (UI만, 기능은 추후)
- [ ] 히어로 섹션: 타이틀 + 서브타이틀
- [ ] AI 추천 입력창: 자연어 입력 + "추천받기" 버튼
- [ ] 구분선: "또는"
- [ ] 커스텀 만들기 버튼
- [ ] 인기 도시 카드: 도쿄, 오사카, 교토, 후쿠오카 (이미지 placeholder)
- [ ] 다크모드 토글 (헤더에 위치)
- [ ] 반응형: 모바일/태블릿/데스크톱
- [ ] Framer Motion 진입 애니메이션

## 화면/인터랙션
- AI 추천 입력 후 "추천받기" 클릭 → `/planner` 이동 (추후 AI 연동)
- "직접 만들기" 클릭 → `/planner` 이동 (빈 상태)
- 도시 카드 클릭 → `/planner?city=tokyo` 식으로 이동
- 다크모드 토글 → 테마 전환

## 라우팅
- `/` → 랜딩 페이지 (이번 구현)
- `/planner` → 플래너 페이지 (placeholder, 추후 구현)

## 데이터 구조
```typescript
interface CityCard {
  id: string
  name: string        // "도쿄"
  nameEn: string      // "Tokyo"
  image: string       // placeholder URL
  description: string // "일본의 수도, 전통과 현대의 조화"
}
```

- [ ] "일본 여행 플래너" 타이틀이 표시되는지
- [ ] AI 입력창이 존재하는지
- [ ] 추천받기 버튼이 존재하는지
- [ ] 직접 만들기 버튼이 존재하는지
- [ ] 4개 도시 카드가 렌더링되는지
